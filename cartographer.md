## Module Purposes

### src/sentinel/parser.py
**Phase:** 1 (Foundation)
**Purpose:** Parse and validate IAM policy JSON
**Responsibilities:**
- JSON structure validation
- Action name syntax checking
- ARN format validation
- Three-tier classification (Tier 1/2/3)
**Dependencies:** database.py
**Dependents:** analyzer.py, self_check.py

### src/sentinel/database.py
**Phase:** 1 (Foundation)
**Purpose:** SQLite interface for IAM actions and resource inventory
**Responsibilities:**
- Database schema creation and migrations
- Action lookup queries
- Resource ARN queries
- Database versioning
**Dependencies:** None (SQLite3 built-in)
**Dependents:** parser.py, analyzer.py, rewriter.py, inventory.py

### src/sentinel/inventory.py
**Phase:** 1 (Foundation - schema), 3 (Complete)
**Purpose:** Manage local resource inventory
**Responsibilities:**
- Resource ARN storage and retrieval
- Resource type categorization
- Placeholder ARN generation
- Inventory queries
**Dependencies:** database.py
**Dependents:** rewriter.py

### src/sentinel/analyzer.py
**Phase:** 2 (Core Analysis)
**Purpose:** Risk analysis and intent mapping
**Responsibilities:**
- Intent-to-access-level translation
- Wildcard detection and scoring
- Privilege escalation detection
- Dangerous permission checking
- Missing companion permission detection
- Cross-statement analysis
**Dependencies:** parser.py, database.py
**Dependents:** rewriter.py

### src/sentinel/rewriter.py
**Phase:** 3 (Policy Generation)
**Purpose:** Generate least-privilege policies
**Responsibilities:**
- Wildcard replacement
- Condition key injection
- Companion permission addition
- Statement reorganization
- Placeholder ARN insertion
**Dependencies:** parser.py, analyzer.py, database.py, inventory.py
**Dependents:** self_check.py

### src/sentinel/self_check.py
**Phase:** 4 (Quality Assurance)
**Purpose:** Validate rewritten policies
**Responsibilities:**
- Re-validation of rewritten policy
- Functional completeness checking
- Tier 2 action exclusion verification
- Assumption statement validation
**Dependencies:** parser.py, database.py
**Dependents:** cli.py

### src/sentinel/cli.py
**Phase:** 5 (Tooling)
**Purpose:** Command-line interface
**Responsibilities:**
- Argument parsing
- File I/O
- Interactive prompts
- Output formatting
- Pipeline orchestration
**Dependencies:** parser.py, analyzer.py, rewriter.py, self_check.py
**Dependents:** None (entry point)

### src/refresh/policy_sentry.py
**Phase:** 5 (Tooling)
**Purpose:** Fetch and import policy_sentry data
**Responsibilities:**
- policy_sentry data retrieval
- Data format conversion
- Database import/merge
**Dependencies:** database.py
**Dependents:** None (utility script)

### src/refresh/aws_docs.py
**Phase:** 5 (Tooling)
**Purpose:** Scrape AWS Service Authorization Reference
**Responsibilities:**
- AWS docs web scraping
- Action extraction
- Database update
**Dependencies:** database.py
**Dependents:** None (utility script)

## Database Schemas

### iam_actions.db

```sql
services(
    service_prefix TEXT PRIMARY KEY,
    service_name TEXT NOT NULL
)

actions(
    action_id INTEGER PRIMARY KEY,
    service_prefix TEXT,
    action_name TEXT,
    access_level TEXT,
    resource_types TEXT,
    condition_keys TEXT,
    UNIQUE(service_prefix, action_name)
)

resource_types(
    resource_type_id INTEGER PRIMARY KEY,
    service_prefix TEXT,
    resource_type TEXT,
    arn_pattern TEXT
)

metadata(
    key TEXT PRIMARY KEY,
    value TEXT
)
```

### resource_inventory.db

```sql
resources(
    resource_id INTEGER PRIMARY KEY,
    service_prefix TEXT,
    resource_type TEXT,
    resource_arn TEXT UNIQUE,
    resource_name TEXT,
    region TEXT,
    account_id TEXT,
    metadata TEXT
)

INDEX idx_service ON resources(service_prefix)
INDEX idx_type ON resources(resource_type)
```

## Conventions

### Naming
- Module files: lowercase with underscores (parser.py, self_check.py)
- Classes: PascalCase (PolicyParser, RiskAnalyzer)
- Functions: lowercase with underscores (validate_policy, map_intent_to_access_level)
- Constants: UPPERCASE with underscores (TIER_1_VALID, ACCESS_LEVEL_READ)

### Type Hints
- All function signatures must have type hints
- Use dataclasses for structured data
- Use typing.Optional for nullable values
- Use typing.List, typing.Dict for collections

### Documentation
- Google-style docstrings for all public functions
- Include Args, Returns, Raises sections
- Document assumptions explicitly

### Error Handling
- Raise specific exceptions (ValueError, KeyError, etc.)
- Include context in exception messages
- Log errors with severity levels

## Gotchas

### SQLite Limitations
- Single writer lock - no concurrent writes
- TEXT type for JSON storage - requires json.loads()
- ROWID auto-increment - use INTEGER PRIMARY KEY

### IAM Action Naming
- Service prefix always lowercase
- Action name always PascalCase
- Separator is colon, not underscore or hyphen

### Resource ARN Formats
- Different services have different ARN structures
- Some actions require multiple ARN formats (S3 bucket vs object)
- Wildcards in ARNs are positional (prefix or suffix, rarely middle)

### Companion Permissions
- Lambda always needs CloudWatch Logs
- VPC resources need ec2 network interface permissions
- KMS-encrypted resources need kms:Decrypt and kms:GenerateDataKey
